## Core value

OpenFang is the only agent OS that fits on weak ARM hardware. The ChaiTex fork adds **self-learning** capabilities (skill self-patching, active memory reasoning about the user, full-text session search) so an autonomous agent can improve its own skills and build a model of its user over time — without losing the footprint advantage that makes pc162 viable.

This is not a port of Hermes Agent System. It is a focused additive extension of OpenFang on three axes where Hermes leads.

## Target hardware

**pc162** — Allwinner Cortex-A7 (4×~1.2 GHz), 1.9 GB RAM, ARM64 (Armbian 26.2.1 / Ubuntu 24.04). OpenFang currently runs there at **116 MB idle RSS** with ~1.2 GB free. Hermes (Python + Node.js) needs 600–900 MB peak and cannot fit. Every design choice in this fork must preserve the pc162 viability constraint.

Secondary target: home2 (Ryzen 7 3700X) for development and headroom.

## Strategic constraints

- **Backward compatible.** Existing 60 bundled skills, 6-layer memory substrate, 40 channel adapters, 14-crate workspace stay working. New features are additive.
- **Security first.** 16 existing security subsystems (taint, prompt-injection scan, signed manifests, sandboxing, etc.) are not weakened. Skill self-patching goes through the same scan pipeline. `skill-manage` itself is protected from agent mutation.
- **No new runtime dependencies.** Reuse bundled SQLite FTS5, reuse agent's LLM driver via `KernelHandle`, no external services for the self-learning loop.
- **pc162-viable.** New code budgets: schema migration completes <30s on SD card, FTS5 search <500ms on typical histories, reasoning at level High <$0.25/query on hosted LLMs (Max gated behind approval).
- **Rust 2021, MSRV 1.75** (per workspace `Cargo.toml`). New crate `openfang-reasoning` uses same edition.
- **One bin, one process.** No microservices. Everything compiles into the single `openfang` binary.

## Roadmap shape

Three phases (see `ROADMAP.md`):

1. **Self-Learning Core** — skill self-patching, FTS5 session search, memory reasoning. 6-8 weeks.
2. **Tool Expansion** — built-in tools 23 → 50+, auto-discovery, cron chaining. 4-8 weeks.
3. **Coordination** — batch agent delegation, orchestration polish. 4-6 weeks.

Phase 1 is the only phase fully designed today. Phases 2-3 will be scoped after Phase 1 lands.

## Out of scope (explicit non-goals)

- Porting Hermes. Cost-benefit was analyzed in [`docs/chaitex/hermes-on-pc162-rust-port.md`](../docs/chaitex/hermes-on-pc162-rust-port.md); rejected.
- Replacing OpenFang's workflow engine, channel adapters, or A2A/OFP protocol stack — those already exceed Hermes.
- Modifying `openfang-cli` (interactive CLI under active independent development by the user).
- Adding a new database. Stays SQLite + bundled.
- Hosting a model locally on pc162 — reasoning calls go to the agent's configured LLM provider (same provider used for normal turns).
