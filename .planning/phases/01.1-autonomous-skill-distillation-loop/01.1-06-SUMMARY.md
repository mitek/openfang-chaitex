# Phase 1.1 Plan 06: Skill-Failure Recording, Patch Proposals, and Memory-Consolidation Nudge Summary

**One-liner:** skill_execute errors feed into SkillFailureTracker with threshold-driven ApprovalRequest routing for protected skills, plus a budget-charged per-agent memory-consolidation background task that persists synthesized facts into UserProfile via load/add/save_profile.

## What Was Built

### Task 1 — record_skill_failure accessor + dispatch interception (SI-01)

Added a `fn record_skill_failure(&self, _skill, _agent, _error_hash: u64) {}` default no-op accessor to the `KernelHandle` trait in `kernel_handle.rs`. This follows the established default-no-op pattern used by `reasoning_engine()`, `skill_registry()`, etc.

In `tool_runner.rs`, the `"skill_execute"` match arm was expanded to capture the result, hash the error message with `DefaultHasher`, and call `kernel.record_skill_failure(skill_name, agent, hash)` on failure.

In `kernel.rs`:
- Added `pub skill_failure_tracker: crate::skill_failure::SkillFailureTracker` field
- Initialized with `SkillFailureTracker::new()` in the construction literal
- Implemented `record_skill_failure` in `impl KernelHandle for OpenFangKernel`: records the failure, checks against `config.distillation.failure_patch_threshold`, calls `propose_skill_patch` and clears the key when threshold is reached
- Added a temporary stub `propose_skill_patch` to allow the crate to compile before Task 2

### Task 2 — propose_skill_patch real implementation (SI-02)

Replaced the stub with a routing implementation:

- Reads `skill_registry` under a sync `RwLock` guard (dropped before any `.await`) to check `manifest.skill.protected: Option<bool>`
- Fail-safe: missing skill OR `None` protected flag treated as `true` (protected)
- **Protected path**: builds a concrete `ApprovalRequest` (`tool_name="skill_distillation_patch"`, `RiskLevel::Low`, `timeout_secs=300`), upgrades the `Weak<OpenFangKernel>` self-reference, and `tokio::spawn`s the `approval_manager.request_approval(req)` call (fire-and-forget) — this avoids cloning the non-Clone `ApprovalManager`
- **Mutable path**: INFO-logs the patch proposal for the agent to act on via `skill_manage`

### Task 3 — Memory-consolidation nudge background task (MC-01/MC-02)

Added to `start_background_agents` after the memory-consolidation block:
```
if config.distillation.consolidation_nudge_hours > 0 {
    tokio::spawn(async move { interval.tick().await; // skip first tick ... })
}
```

Implemented `run_memory_consolidation_nudge` as an async method on `Arc<Self>`:
1. Guards against missing `reasoning_engine` (INFO-skips)
2. Snapshots agent list (id + name) before any `.await` — no registry guard held across the call
3. For each agent: builds a `ReasoningQuery { level: Medium, max_facts: 30 }` and calls `engine.reason(req).await`
4. On budget-exceeded or other `ReasoningError`: `tracing::info!` and `continue`
5. Filters results with `confidence < 0.7` or answer == "NONE"
6. **MC-01 PERSIST**: calls `load_profile`, `add_fact` with `FactSource::MemoryReason { level: "medium" }`, `save_profile` — identical to the `memory_conclude` tool path
7. Write failures are WARN-logged but do not abort the per-agent loop
