## Accomplishments

- Enforces 20-event hard cap per key (pc162 memory constraint from RESEARCH.md Pitfall 5)

## Decisions Made

- Used `record_failure_at` for test-friendly timestamp injection following the cron.rs time-window pattern (which also uses chrono::Utc for windowing rather than std::time::Instant)
- TTL decay on insert path: stale entries are purged before the new event is pushed. This means `failures_in_window` is always correct without a separate filter, and stale entries cannot accumulate indefinitely between reads
- `DashMap` confirmed already a direct dep of `openfang-kernel` (line 22 of Cargo.toml) — no Cargo.toml changes needed
- Both `dashmap` and `chrono` confirmed as direct kernel deps — plan was correct, no dependency additions required
