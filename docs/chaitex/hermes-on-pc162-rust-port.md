# Анализ: Hermes Agent System на pc162 и перспективы Rust-порта

**Дата**: 2026-06-06  
**Задача**: Оценить возможность запуска Hermes Agent System на устройстве pc162. Если невозможно — исследовать целесообразность портирования на Rust.

## 2. Требования Hermes Agent System

### Текущий профиль (home2: Ryzen 7 3700X, 39 GB RAM)

| Компонент | Размер/потребление |
|-----------|-------------------|
| **Python-процесс (gateway, idle)** | 432 MB RSS |
| **node_modules** | 69 MB на диске |
| **~/.hermes/ (все данные)** | 2.1 GB |
| **Python** | 3.14.3 |
| **Node.js** | >= 20.0.0 (v25.4.0) |
| **Архитектура** | x86_64 |

### Ключевые архитектурные факты

- **Основной цикл**: `run_agent.py` — ~12K строк Python, интерпретируемый
- **Gateway**: Python + Node.js (Ink/React для TUI)
- **Инструменты**: Python, авто-обнаружение через `tools/registry.py`
- **Плагины**: Python (model-providers, memory, context_engine, kanban)
- **Memory/Sessions**: SQLite FTS5
- **Системные требования**: Python 3.11+, Node.js 20+, SQLite

### Реальное потребление под нагрузкой

В idle — 432 MB. При активной работе (модельный вызов + инструменты + парсинг ответа):
- Контекстное окно в памяти: 50-200 MB (зависит от длины диалога)
- Инструменты (terminal, browser, delegate_task): ещё 50-150 MB на процесс
- Пиковое потребление: **600-900 MB**

## 3. Совместимость: pc162 × Hermes

### Вердикт: ❌ Несовместимо

| Фактор | Проблема | Детали |
|--------|----------|--------|
| **RAM** | Критическая | Hermes требует 600-900 MB пиково. На pc162 свободно 1.2 GB, но с учётом системных нужд и уже запущенного OpenFang — запас около 500 MB. Этого категорически мало. |
| **CPU** | Критическая | Cortex-A7 @ 1.2 GHz против Ryzen 7 3700X. Python-интерпретация на Cortex-A7 будет в 10-20× медленнее. Модельный вызов сам по себе — API latency, но обработка ответа, парсинг JSON, инструменты — все на CPU. |
| **Диск** | Терпимо | 9.2 GB свободно. Хватит на установку (2-3 GB Python + Node.js + зависимости). Но SD-карта — медленная. |
| **Node.js на ARM** | Терпимо | Node.js собирается под aarch64. Бинарники есть. Но TUI (Ink/React) — лишняя нагрузка. |
| **Python-зависимости** | Проблема | Часть пакетов может не собраться под aarch64 (нативные расширения C). |

### Что если убрать XFCE desktop?

Без графики освободится ~350 MB. Пиково доступно станет ~850 MB. Но:

- Этого всё равно на грани для Hermes (600-900 MB пиково)
- Cortex-A7 остаётся узким местом — время ответа агента будет секунд 30-60 вместо 2-5
- OpenFang + Hermes на одной машине — конкуренция за CPU и память

**Даже в минимальной конфигурации (без TUI, без XFCE, с урезанными инструментами) — не рекомендуется.**

## 4. Портирование на Rust: анализ

### Что такое «портировать Hermes на Rust»

Это не портирование. Это **переписывание** ~50K+ строк Python + Node.js на Rust. По сути — новый проект, использующий дизайн-решения Hermes.

### Breakdown по компонентам

| Компонент | Сложность переписывания | Примечание |
|-----------|------------------------|------------|
| **Основной цикл (run_agent.py)** | Высокая (12K LOC) | Конечный автомат, управление контекстом, retry-логика |
| **Gateway (мессенджеры)** | Очень высокая | 20+ платформ, каждая со своим API. Python-экосистема для этого зрелая, Rust — нет. |
| **Инструменты** | Высокая | 50+ инструментов. Многие завязаны на Python-экосистему (lxml, pillow, openpyxl). |
| **Memory/Sessions (SQLite FTS5)** | Низкая | SQLite в Rust — зрелые биндинги (rusqlite). |
| **Плагинная система** | Средняя | Rust не имеет нативного plugin-механизма как Python. WASM или dynamic loading. |
| **TUI (Ink/React)** | Очень высокая | В Rust нет аналога Ink. Ratatui — терминальный, не React. |
| **Model providers** | Средняя | HTTP-клиенты. В Rust — зрелые (reqwest). |
| **Cron/планировщик** | Низкая | Есть зрелые crate'ы. |

### Оценка усилий

- **Полное переписывание**: 12-18 человеко-месяцев опытного Rust-разработчика
- **Результат**: другая система. Многие возможности Hermes построены на Python-экосистеме и невоспроизводимы в Rust за разумное время (gateway-платформы, некоторые инструменты)
- **Производительность**: выигрыш в RAM (50-100 MB вместо 400+ MB) и CPU (в 5-20× быстрее на том же железе)

### А что реально?

Есть **два разных сценария**:

#### А) Легковесный агент на Rust для слабых устройств (реально)

Не порт Hermes, а новый проект с ограниченной функциональностью:

- Основной цикл агента (конечный автомат, вызовы API)
- 5-10 критичных инструментов (shell, file, web)
- Один-два мессенджера (Telegram, Feishu)
- SQLite-память
- Без TUI, без 50 инструментов, без плагинов
- Целевое потребление: **30-50 MB RAM в idle**

Это реально сделать за 2-3 человеко-месяца. На pc162 заработает.

#### Б) Полный порт Hermes на Rust (нет)

Экономически нецелесообразно:

- Стоимость разработки сопоставима с 1.5-2 годами работы
- Часть функциональности неизбежно потеряется (gateway-платформы, плагины)
- Python-экосистема для AI-агентов развивается быстрее, чем Rust
- Основной потребитель Hermes (Дмитрий на home2) не имеет проблем с ресурсами

## 5. Рекомендация

### Краткосрочно (сейчас)

**Не запускать Hermes на pc162.** Использовать pc162 как:
- OpenFang-хост (уже работает)
- Лёгкий jump-хост для Tailscale-сети
- Вспомогательное устройство для простых cron-задач

Hermes продолжает работать на home2 (Ryzen 7, 39 GB RAM) — это правильная среда для него.

### Среднесрочно (3-6 месяцев)

Если нужен агент на слабом ARM-устройстве — написать легковесного агента на Rust:

- **Не порт Hermes**, а специализированный агент с узкой функциональностью
- Цель: 30-50 MB RAM, запуск на ARM Cortex-A7/A53
- Подход: выделить минимальное ядро (агентский цикл + инструменты + один мессенджер)
- Референс: OpenFang уже написан на Rust, запущен на pc162, потребляет 116 MB

### Ключевой архитектурный вывод

**Hermes и «агент на pc162» — это два разных класса систем:**

| | Hermes (home2) | Гипотетический Rust-агент (pc162) |
|---|---|---|
| **Роль** | Оркестратор + архитектор | Специалист/worker |
| **RAM** | 600-900 MB | 30-50 MB |
| **Язык** | Python + Node.js | Rust |
| **Инструменты** | 50+ | 5-10 |
| **Мессенджеры** | 20+ | 1-2 |
| **Плагины** | Да | Нет |

Это не «порт». Это другой продукт для другого класса устройств.

## 6. Если всё-таки делать Rust-агента

### Минимальный viable стек

```rust
// Cargo.toml
[dependencies]
tokio = { version = "1", features = ["full"] }     // async runtime
reqwest = { version = "0.12", features = ["json"] } // HTTP client (LLM API)
rusqlite = { version = "0.31", features = ["bundled"] } // SQLite memory
serde = { version = "1", features = ["derive"] }     // JSON
serde_json = "1"
teloxide = "0.13"  // или feishu-sdk для мессенджера
```

### Архитектура ядра (псевдокод)

```rust
struct Agent {
    model: ModelClient,        // OpenAI-совместимый HTTP
    tools: Vec<Box<dyn Tool>>, // shell, file, web_search
    memory: SqliteMemory,      // сессии, факты
    messenger: Messenger,      // один мессенджер
}

impl Agent {
    async fn run_loop(&mut self, user_message: String) -> String {
        loop {
            let response = self.model.chat(&self.conversation).await?;
            if let Some(tool_calls) = response.tool_calls {
                for call in tool_calls {
                    let result = self.execute_tool(call).await;
                    self.conversation.push(result);
                }
            } else {
                return response.content;
            }
        }
    }
}
```

### Оценка ресурсов на pc162

| Компонент | RAM |
|-----------|-----|
| Бинарник Rust (статический) | 15-20 MB |
| Контекстное окно (10K токенов) | 5-10 MB |
| SQLite memory | 5-10 MB |
| HTTP-клиент | 5 MB |
| **Итого idle** | **30-45 MB** |
| Под нагрузкой | 60-80 MB |

На pc162 с 1.2 GB доступной памяти — **запустится уверенно, с запасом 1+ GB**.
